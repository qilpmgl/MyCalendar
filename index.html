云

Pages 自动发布

仓库 push 之后，GitHub Pages 会重新部署。你只要维护数据文件，网页入口就一直可访问。

使用方式

如果你准备把这个站点作为固定入口，下面这三个动作最常用。

1. 打开 GitHub 仓库设置中的 Pages，把部署来源设为 main 分支和根目录 /(root)。
2. 以后运行脚本重新生成 chinese_almanac.ics，提交并 push，网页下载内容就会更新。
3. 部署完成后，访问 https://qilpmgl.github.io/MyCalendar 就能看到这个页面。
